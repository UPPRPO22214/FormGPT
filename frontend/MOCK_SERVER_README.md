# Мок-сервер для тестирования фронтенда

Этот мок-сервер имитирует API бэкенда для тестирования фронтенда без реального бэкенда.

## Установка зависимостей

```bash
npm install
```

## Запуск мок-сервера

В отдельном терминале запустите:

```bash
npm run dev:mock
```

Или напрямую:

```bash
node mock-server.js
```

Мок-сервер будет доступен на `http://localhost:8081`

## Запуск фронтенда

В другом терминале запустите:

```bash
npm run dev
```

Фронтенд будет доступен на `http://localhost:3000`

## Реализованные эндпоинты

- `POST /auth/register` - Регистрация (проверяет уникальность email)
- `POST /auth/login` - Авторизация (принимает любой пароль для существующего email)
- `GET /users/me` - Получение профиля (требует Bearer token)
- `PUT /users/me` - Обновление профиля (требует Bearer token)
- `PUT /users/me/password` - Смена пароля (требует Bearer token)

## Особенности мок-сервера

- Хранит пользователей в памяти (данные теряются при перезапуске)
- Генерирует простые токены для авторизации
- Принимает любой пароль для существующих пользователей
- Проверяет уникальность email при регистрации
- Валидирует минимальную длину пароля (6 символов)

## Тестирование

1. Зарегистрируйте нового пользователя через форму регистрации
2. Войдите с теми же данными
3. Проверьте профиль - данные должны отображаться
4. Попробуйте редактировать профиль
5. Попробуйте сменить пароль

## Примечание

Мок-сервер работает только для тестирования UI. Для реальной работы нужен настоящий бэкенд.
